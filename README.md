# 口袋冰箱 · Pocket Fridge

一个以手机端为核心、使用 Three.js 实现的低多边形 3D 互动冰箱。

## 在线体验

**<https://dreaminmaster.github.io/2fridge/>**

## 当前版本

- 手机端直接使用单指旋转与双指缩放，不再显示多余的放大、缩小和重置视角按钮
- 保留高清抗锯齿和 Retina 清晰度，手机像素比上限为 `1.8`
- 改为按需渲染：静止时停止持续满帧重绘，操作、柜门动画和食物动画发生时才绘制
- 阴影保持高质量，但使用静态缓存，不在每一次相机旋转中重复计算
- Safari 地址栏高度变化不会再重置用户当前视角
- 库存采用事务式添加：食物 3D 模型创建成功后，才正式占用冰箱容量
- 所有 48 种食物模型均有自动运行时测试，防止单个模型失败造成“看不见但占满容量”
- 每种食物都会测量真实三维包围盒，自动居中、缩放并贴合层板、冷冻底板或门架
- 每个槽位都有独立的宽度、高度与深度安全范围，避免相邻食物穿模
- 冷冻底层会根据较小的垂直空间自动缩小高食物，避免穿过上方层板
- 层板新增前沿挡边、后沿止挡，门架新增左右护栏与前挡条
- 冰箱内部增加冷冻底部内衬、灯罩和温控面板等细节
- 旧库存会自动清理未知食物、重复槽位和无效槽位，并迁移到 v3 数据结构
- 上下柜门独立开合，可以观察冰箱深处和柜门内侧
- 冰箱柜体与层板具有真实深度，同一层分为前排与后排
- 新食物优先进入可见的前排，前排放满后再使用后排
- 手机端添加食物后自动收起面板、打开对应柜门，并短暂标记新食物位置
- 冷藏区 24 个位置、冷冻区 16 个位置、门内 12 个位置，总容量 52 件
- 每件食材拥有独立 `instanceId`、存放区域、层号、前后排和固定槽位
- 轻点 3D 食材可查看所在层与前后排，并可单独移除
- 48 种基础食材，覆盖肉类、海鲜、冷冻食品、蔬菜、水果、乳制品、主食、饮料与调味料
- 手机端底部抽屉和桌面端右侧抽屉使用不同布局
- 本地库存持久化，刷新后继续保留

## 三维储物结构

冷藏和冷冻不再使用单一平面网格，而是：

```text
每一层
├─ 前排：4 个位置（优先使用）
└─ 后排：4 个位置（具有独立纵深）
```

冷藏区共有 3 层，冷冻区包含一层中间层板和一层底部内衬。前四件食物优先摆在当前层前排，随后使用后排。前后排拥有独立的深度安全范围，左右相邻位置也保留碰撞间距。旋转冰箱后可以观察后排内容，但后排食物不会为了显示而悬空抬高。

## 物理摆放系统

食物摆放不再依赖统一的中心坐标。每次创建食物模型时，系统会：

1. 计算模型所有几何体形成的真实三维包围盒；
2. 将模型按照底面重新归一化，使最低点成为稳定支撑点；
3. 根据当前槽位允许的最大宽度、高度和深度自动缩放；
4. 将最低点贴到层板顶面、冷冻底板顶面或门架底板顶面；
5. 保留少量安全间隙，避免与层板表面闪烁或穿插。

因此罐头、蔬菜、肉条、杯子和瓶子即使原始重心与高度完全不同，也会自然落在承托面上。冷冻底层的垂直空间小于中间层板，模型会根据实际空间自动缩放，而不是穿过上层。

## 高清移动端渲染策略

移动端不通过降低画质换取流畅度：

- WebGL 保留抗锯齿、高精度着色和最高 `1.8` 像素比
- 渲染循环从永久 60 FPS 改成事件驱动的按需帧
- 手指拖动期间即时更新，松手后不再空耗 GPU
- 柜门和冰箱阴影使用高质量贴图，但仅在场景结构真正变化后更新
- 食物小模型不单独生成昂贵的动态阴影
- Safari 工具栏导致的高度变化只更新画布尺寸，不重置相机
- 手机端禁用 OrbitControls 惯性拖尾，手指与画面保持直接同步

## 一致性与容错

添加食物分成两个阶段：

1. 预留一个可用槽位并创建、测量、摆放 3D 模型；
2. 模型完整进入场景后，才提交库存记录。

任何模型创建或摆放失败都不会占用容量。页面启动时也会自动修复旧数据中的未知食物、重复槽位和无效槽位。如果已有库存中的某个模型仍无法显示，系统会将该无效记录清理，而不是继续让它占据容量。

## 面向未来菜谱系统的数据结构

库存条目结构：

```js
{
  instanceId: "...",
  foodId: "eggs",
  ingredientKey: "egg",
  zone: "fridge",
  slot: 0,
  quantity: 1,
  unit: "盒",
  addedAt: "..."
}
```

公开菜谱上下文：

```js
window.pocketFridge.getRecipeContext()
```

返回标准食材键、数量、单位、存放区域和菜谱标签。后续连接规则菜谱库、数据库或 AI 菜谱生成时，不需要重写 3D 冰箱与库存层。

## 代码结构

```text
src/
├─ data/
│  └─ foodCatalog.js       食材、分类、容量及菜谱元数据
├─ domain/
│  ├─ inventoryStore.js    事务库存、容量、槽位、迁移与自修复
│  └─ recipeContext.js     菜谱系统标准数据出口
├─ scene/
│  ├─ createFridge.js      深柜体、层板、挡边、门架与内部细节
│  ├─ createFoodModel.js   Low-poly 食物模型
│  ├─ foodPlacement.js     包围盒测量、自动落地、缩放与安全范围
│  ├─ fridgeLayout.js      支撑面、前后排、门架与槽位包围盒
│  └─ sceneController.js   按需渲染、手势、点击和动画
├─ ui/
│  ├─ foodIcon.js          食材界面图标
│  └─ uiController.js      添加、容量、库存与移除
├─ main.js
├─ style.css
└─ mobile-performance.css  手机端触控与界面优化
```

## 本地运行

```bash
npm install
npm run dev
```

测试与生产构建：

```bash
npm test
npm run build
npm run preview
```

GitHub Actions 会在每次推送到 `main` 后自动测试全部食物模型、物理落地、槽位安全范围、库存一致性和项目结构，随后构建并发布到 GitHub Pages。
