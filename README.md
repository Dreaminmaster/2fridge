# 口袋冰箱 · Pocket Fridge

一个以手机端为核心、同时适配电脑端的 Three.js 低多边形 3D 互动冰箱。

## 在线体验

**<https://dreaminmaster.github.io/2fridge/>**

## 当前版本

- 手机端直接使用单指旋转与双指缩放，不显示多余的视角按钮
- 电脑端可在旋转模式与“移动冰箱”模式之间切换
- 移动模式下用鼠标拖动冰箱调整画面位置，恢复视角会同时恢复冰箱位置
- 移除了远距离场景雾化，缩小查看时仍保持清晰和正常对比度
- 冰箱会根据真实三维底部边界与当前缩放比例自动贴地，避免桌面宽屏下底部穿入地板
- 保留高清抗锯齿和 Retina 清晰度，手机像素比上限为 `1.8`
- 改为按需渲染：静止时停止持续满帧重绘，操作和动画发生时才绘制
- 阴影保持高质量，但使用静态缓存，不在每一次相机旋转中重复计算
- Safari 地址栏高度变化不会重置用户当前视角
- 库存采用事务式添加：食物 3D 模型创建成功后，才正式占用容量
- 所有 48 种食物模型均有自动运行时测试
- 食物会测量真实三维包围盒，自动居中、缩放并贴合层板、冷冻底板或门架
- 每个槽位都有独立的宽度、高度与深度安全范围，避免相邻食物穿模
- 层板拥有前沿挡边、后沿止挡，门架拥有左右护栏与前挡条
- 冰箱内部包括冷冻底部内衬、灯罩和温控面板等细节
- 上下柜门独立开合，可以观察冰箱深处和柜门内侧
- 冰箱柜体与层板具有真实深度，同一层分为前排与后排
- 新食物优先进入可见前排，前排放满后再使用后排
- 手机端食材抽屉支持连续添加，添加一件后不会自动关闭
- 连续添加时保留当前分类、搜索条件和列表滚动位置
- 抽屉内显示刚刚添加的食材，并可精确撤销上一次添加
- 冷藏区 24 个位置、冷冻区 16 个位置、门内 12 个位置，总容量 52 件
- 每件食材拥有独立 `instanceId`、存放区域、层号、前后排和固定槽位
- 轻点 3D 食材可查看所在层与前后排，并可单独移除
- 48 种基础食材覆盖肉类、海鲜、冷冻食品、蔬菜、水果、乳制品、主食、饮料与调味料
- 本地库存持久化，刷新后继续保留

## 电脑端视角与移动

默认状态下，鼠标拖动用于旋转冰箱，滚轮用于缩放。右侧的 `✥` 按钮可以切换“移动冰箱”模式：

1. 点击 `✥`，按钮变为绿色；
2. 按住鼠标左键拖动，将冰箱在地面范围内重新摆放；
3. 再次点击 `✥` 返回旋转模式；
4. 点击恢复视角，摄像机与冰箱位置一起恢复默认状态。

移动只改变冰箱在场景中的位置，不会改变库存、柜门状态或食材槽位。冰箱的纵向高度由实际包围盒自动计算，所以无论响应式缩放、开门缩小或手动移动，底部都会保持贴合地面。

## 食物模型细节

现有基础模型增加了第二层低多边形识别细节，并继续由物理摆放系统统一测量：

- 肉类增加托盘、脂肪纹路、煎纹或配菜点缀
- 罐头和饮料增加标签、顶盖与拉环
- 牛奶和果汁纸盒增加正面标识与瓶盖
- 瓶装调味品增加标签、瓶颈和封口圈
- 鸡蛋增加蛋托
- 黄油、豆腐增加包装带和标识
- 奶酪增加孔洞层次
- 蔬果增加高光、叶片、斑点、根部或绑带
- 面包增加割纹，米饭增加碗沿和筷子

这些装饰会在计算包围盒之前加入，因此模型变丰富后仍会自动适配槽位，不会重新引入穿模、悬浮或超出冰箱的问题。

## 连续添加体验

手机端打开“添加食材”后，抽屉会一直保持打开，直到用户主动点击“完成”。每次成功添加都会：

1. 保留当前搜索词、分类和滚动位置；
2. 高亮刚刚选择的食材卡片；
3. 在抽屉内显示本次添加结果；
4. 提供“撤销”按钮，精确移除最后一次添加；
5. 自动打开对应柜门并标记食材在冰箱中的位置。

## 三维储物结构

```text
每一层
├─ 前排：4 个位置（优先使用）
└─ 后排：4 个位置（具有独立纵深）
```

冷藏区共有 3 层，冷冻区包含一层中间层板和一层底部内衬。前后排拥有独立的深度安全范围，左右相邻位置保留碰撞间距。旋转冰箱后可以观察后排内容，但后排食物不会为了显示而悬空抬高。

## 物理摆放系统

每次创建食物模型时，系统会：

1. 计算模型全部几何体形成的真实三维包围盒；
2. 将模型按照底面重新归一化；
3. 根据槽位允许的最大宽度、高度和深度自动缩放；
4. 将最低点贴到层板顶面、冷冻底板顶面或门架底板顶面；
5. 保留少量安全间隙，避免表面闪烁或穿插。

任何模型创建或摆放失败都不会占用容量。页面启动时也会自动修复未知食物、重复槽位和无效槽位。

## 面向未来菜谱系统的数据结构

库存条目结构：

```js
{
  instanceId: "...",
  foodId: "eggs",
  ingredientKey: "egg",
  zone: "fridge",
  slot: 0,
  quantity: 1,
  unit: "盒",
  addedAt: "..."
}
```

公开菜谱上下文：

```js
window.pocketFridge.getRecipeContext()
```

返回标准食材键、数量、单位、存放区域和菜谱标签。后续连接规则菜谱库、数据库或菜谱生成模块时，不需要重写 3D 冰箱与库存层。

## 代码结构

```text
src/
├─ data/
│  └─ foodCatalog.js       食材、分类、容量及菜谱元数据
├─ domain/
│  ├─ inventoryStore.js    事务库存、容量、槽位、迁移与自修复
│  └─ recipeContext.js     菜谱系统标准数据出口
├─ scene/
│  ├─ createFridge.js      深柜体、层板、挡边、门架与内部细节
│  ├─ createFoodModel.js   Low-poly 食物基础模型
│  ├─ foodDetails.js       包装、纹理和识别细节增强
│  ├─ foodPlacement.js     包围盒测量、自动落地、缩放与安全范围
│  ├─ fridgeLayout.js      支撑面、前后排、门架与槽位包围盒
│  └─ sceneController.js   渲染、旋转、缩放、移动、点击和动画
├─ ui/
│  ├─ foodIcon.js          食材界面图标
│  └─ uiController.js      连续添加、撤销、容量、库存与移除
├─ main.js
├─ style.css
├─ mobile-performance.css
└─ desktop-polish.css
```

## 本地运行

```bash
npm install
npm run dev
```

测试与生产构建：

```bash
npm test
npm run build
npm run preview
```

GitHub Actions 会在每次推送到 `main` 后自动测试食物模型、物理落地、槽位安全范围、电脑端交互、连续添加、库存一致性和项目结构，随后构建并发布到 GitHub Pages。
