# 口袋冰箱 · Pocket Fridge

一个以手机端为核心、同时适配电脑端的 Three.js 低多边形 3D 互动冰箱。

## 在线体验

**<https://dreaminmaster.github.io/2fridge/>**

## 当前版本

- 手机端使用单指旋转与双指缩放，不显示多余视角按钮
- 电脑端鼠标左键拖动旋转，Mac 触控板双指滑动平移视角，捏合缩放
- 鼠标右键拖动也可以平移相机视角，冰箱模型本身始终固定并自动贴地
- 移除远距离场景雾化，拉远查看时仍保持清晰和正常对比度
- 冷藏区正式增加第四层：三块层板加底部内衬，共 32 个位置
- 冷藏区 32 个位置、冷冻区 16 个位置、门内 12 个位置，总容量 60 件
- 阴影只由冰箱外壳和柜门等大轮廓投射，柜门动画中连续更新，避免结束时突然跳变
- 保留高清抗锯齿和 Retina 清晰度，手机像素比上限为 `1.8`
- 使用按需渲染，静止时停止持续满帧重绘
- 冰箱根据真实三维底部边界与当前缩放比例自动贴地
- 库存采用事务式添加：3D 模型成功进入场景后才正式占用容量
- 48 种现有食物继续完善模型细节，不增加新的食材种类
- 食物会测量真实三维包围盒，自动居中、缩放并贴合承托面
- 每个槽位具有独立宽度、高度和深度安全范围，避免悬浮与穿模
- 上下柜门独立开合，冷藏和冷冻同一层均分为前排与后排
- 手机端食材抽屉支持连续添加，并保留搜索、分类和滚动位置
- 每件食材拥有独立 `instanceId`、存放区域、层号、前后排和固定槽位
- 轻点 3D 食材可查看位置并单独移除
- 本地库存持久化，刷新后继续保留

## 电脑端视角操作

电脑端改变的是摄像机，而不是冰箱在房间中的位置：

1. 鼠标左键拖动：旋转视角；
2. Mac 触控板双指滑动：上下左右平移整个画面；
3. 触控板捏合：缩放；
4. 鼠标右键拖动：平移视角；
5. 鼠标中键或页面视角按钮：缩放；
6. 恢复视角：相机位置、观察中心和缩放距离回到默认状态。

视角平移有合理边界，避免把冰箱完全移出画面。移动相机不会改变库存、柜门状态或食物槽位。

## 冷藏第四层

冷藏室内部现在包含四个真实承托面：

```text
冷藏区
├─ 第 1 层：前排 4 + 后排 4
├─ 第 2 层：前排 4 + 后排 4
├─ 第 3 层：前排 4 + 后排 4
└─ 第 4 层（底部内衬）：前排 4 + 后排 4
```

底部区域增加独立内衬、前挡边、后挡边和侧轨，不再只是视觉上的空区域。原有 24 个冷藏槽位编号保持不变，新增加的第四层使用后续 8 个槽位，因此旧库存可以继续加载。

## 光影与性能

光影优化不依赖降低整体画质：

- 阴影贴图仍保持高分辨率和柔和过滤；
- 食物、把手、层板标签等小几何体不再重复参与昂贵阴影绘制；
- 冰箱外壳与柜门保留动态阴影；
- 柜门运动期间以轻量阴影投射集合连续更新，避免动画结束后阴影突然跳动；
- 相机旋转和平移不会无意义重算世界阴影；
- 静止后停止渲染循环，降低 GPU 空耗。

## 现有食物模型细节

不新增食物种类，只继续增强现有模型：

- 肉类保留托盘、脂肪纹路、煎纹和配菜点缀
- 香肠增加两端扎口
- 罐头和饮料具有标签、顶盖与拉环
- 瓶装饮料和调味品增加瓶身环纹、标签与封口细节
- 酸奶和冰淇淋增加铝膜与揭盖片
- 豆腐增加切块线，奶酪增加外沿，蘑菇增加菌褶
- 西红柿、苹果、柠檬增加叶片，洋葱增加根须
- 生菜和白菜增加叶脉，面包增加割纹和表面屑点
- 香蕉增加末端细节，米饭保留碗沿和筷子

全部细节会在包围盒计算前加入，因此模型完善后仍会自动适配槽位，不重新引入穿模或悬浮。

## 连续添加体验

手机端打开“添加食材”后，抽屉会保持打开，直到用户主动点击“完成”。每次成功添加都会保留当前搜索词、分类和滚动位置，高亮刚刚选择的食材，显示存放结果，并允许撤销最后一次添加。

## 三维储物结构

```text
每一层
├─ 前排：4 个位置（优先使用）
└─ 后排：4 个位置（具有独立纵深）
```

前后排拥有独立的深度安全范围，左右相邻位置保留碰撞间距。旋转或平移视角后可以观察后排内容，但后排食物不会为了显示而悬空抬高。

## 面向未来菜谱系统的数据结构

库存条目结构：

```js
{
  instanceId: "...",
  foodId: "eggs",
  ingredientKey: "egg",
  zone: "fridge",
  slot: 0,
  quantity: 1,
  unit: "盒",
  addedAt: "..."
}
```

公开菜谱上下文：

```js
window.pocketFridge.getRecipeContext()
```

返回标准食材键、数量、单位、存放区域和菜谱标签。后续连接规则菜谱库、数据库或菜谱生成模块时，不需要重写 3D 冰箱与库存层。

## 代码结构

```text
src/
├─ data/
│  └─ foodCatalog.js       食材、分类、容量及菜谱元数据
├─ domain/
│  ├─ inventoryStore.js    事务库存、容量、槽位、迁移与自修复
│  └─ recipeContext.js     菜谱系统标准数据出口
├─ scene/
│  ├─ createFridge.js      四层冷藏、柜体、门架、阴影投射与内部细节
│  ├─ createFoodModel.js   Low-poly 食物基础模型
│  ├─ foodDetails.js       第一层包装和识别细节
│  ├─ foodPolish.js        现有食物第二层轮廓与材质细节
│  ├─ foodPlacement.js     包围盒测量、自动落地、缩放与安全范围
│  ├─ fridgeLayout.js      四层支撑面、前后排、门架与槽位包围盒
│  └─ sceneController.js   按需渲染、相机旋转、平移、缩放和动画
├─ ui/
│  ├─ foodIcon.js          食材界面图标
│  └─ uiController.js      连续添加、撤销、容量、库存与移除
├─ main.js
├─ style.css
├─ mobile-performance.css
└─ desktop-polish.css
```

## 本地运行

```bash
npm install
npm run dev
```

测试与生产构建：

```bash
npm test
npm run build
npm run preview
```

GitHub Actions 会在每次推送到 `main` 后自动测试食物模型、四层物理落地、槽位安全范围、电脑端相机交互、连续添加、库存一致性和项目结构，随后构建并发布到 GitHub Pages。
